What is AREA ?

 

AREA(for Action Reaction) is about implementing a business application similar to IFTTT and/or Zapier.

This application allow you to manage efficiently your activities through many services interractions configuration.

 

Example:

Plan an activity(play a song) -> Play the song in spotify

 

 

How to build the project

 

Each part of the project is launched with Docker.

Run the following command to launch the docker images building:

Docker build .

 

Run it in the following folders:

area_backend/ : accessible on localhost/8080

client_mobile/area_mobile/ : accessible on localhost:8080/area_mobile.apk

client_web/area/ : accessible on localhost/8081

 

 

APIs used

 

This is the list of APIs used to implement services:

Project architecture

The project has 3 main parts

 

The application server

(area_backend/)

Based on flask.

It's the main part of the project architecture. It offers services implemented to the web & mobile clients through a REST API.

 

The web client

(web_client/)

Developed with react js.

The web client displays screens and forward requests from the user to the application server.

 

The mobile client

(mobile_client/)

Developed with react native.

The mobile client is avalaible on android and ios. Like the web client, it displays screens and forward requests to the application server.

Server endpoints

 

You can also run the server only locally, to access apis endpoints, thanks to swagger.

In area_backend/

Run:

pip install -r requirements.txt

flask run

 

Once the server launched, go to this page:

  • localhost/5000/swagger

There the documentation about our apis endpoints will be available.

The swagger json documentation file is located at area_backend/static/
